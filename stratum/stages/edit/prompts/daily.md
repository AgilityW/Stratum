# 日报生成指令

## 任务

根据以下文章，撰写一份 {{ date_line }} 的{{ domain_title }}。

## 输出格式（严格遵循）

```
### 今日要点

2-3句摘要，概括今日最重要的动态和趋势。

---

### 第一条新闻标题

新闻正文段落。提供深度分析，不只复述事实。

*source1, source2 · YYYY年M月D日*

---

### 第二条新闻标题

...

---

### 【边缘信号】弱相关或弱信号标题

弱信号正文段落。说明它为什么值得观察，也说明为什么暂不提升为主线判断。

*source1, source2 · YYYY年M月D日*

---

### 关注

- 值得持续跟踪但尚未形成独立新闻的事件
- 正在酝酿的技术或产业变化
- 潜在风险信号

---

### 反向信号

- 与主流叙事相悖的数据或观点
- 可能被市场忽视的负面信号
- 需要警惕的乐观偏差

---

*由 AI Agent 自动生成 · 每日 7:30 CST*
```

## 要求

1. 从 {{ article_count }} 篇文章中选出总计 {{ min_items }}-{{ max_items }} 条新闻写成简报；其中主线新闻 {{ main_min_items }}-{{ main_max_items }} 条，另加边缘信号 {{ edge_min_items }}-{{ edge_max_items }} 条，这些数量必须保证
2. 优先覆盖集群化文章，其次是有增量价值的独立文章
3. **严格遵循上方输出格式**——`### 今日要点` → `---` → N条 `### 新闻标题` → `---` → `### 关注` → `### 反向信号`
4. 每条新闻必须引用来源和日期 — **source 必须严格来自下方“来源索引”或文章数据中的来源字段，禁止使用训练数据或外部知识中的其他来源**
5. 每个条目用 `### ` 开头，`---` 分隔
6. 今日要点：2-3句精准摘要，不罗列标题
7. 关注：3项 bullet points（`- ` 开头）
8. 反向信号：3项 bullet points（`- ` 开头），必须包含与主流叙事相悖的信号
9. 其中 {{ edge_min_items }}-{{ edge_max_items }} 条必须是弱相关或弱信号，标题必须以 `【边缘信号】` 开头；这类条目用于覆盖玻璃硬盘、AI 公司投资、存储厂商入股 Anthropic/模型公司、个股/模组厂、论坛/会议侧面信号、尚未形成主线判断的产业链小信号
10. `【边缘信号】` 条目必须说明“为什么值得观察”和“为什么暂不提升为主线判断”，不能用来替代 HBM、价格、产能、国产替代等主线新闻
11. 存储厂商投资 AI 模型公司、个股估值、市值叙事、模组厂单月营收、非半导体存储介质等，默认归为 `【边缘信号】`；只有当来源明确给出供货合同、客户认证、产能承诺、价格变化或技术量产节点时，才能升为主线新闻

## 特别关注

- 跨天事件线（cluster 标记了 thread_label 的）→ 标注"持续追踪"
- 国内存储厂商（CXMT/YMTC）→ 从叙事转向业务判断：认证状态、良率、出货量、客户导入
- 价格变动 → 必须解释传导路径

## 多语言来源要求（CRITICAL）

- 每条尽量引用一个非中文来源，但**来源行不要标注语言标签**，例如不要写 `[en]` 或 `[zh-CN]`
- 不要只覆盖中文媒体——英文、日文、韩文来源必须均匀出现
- 如果同一事件在多语言区都有报导，优先用不同语言来源交叉引用
- 来源行格式：`*source1, source2 · 2026年M月D日*`
- source 用下方来源索引中的字符串；不要自行翻译、补全或替换来源名
