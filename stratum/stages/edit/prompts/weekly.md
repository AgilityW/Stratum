# 周报生成指令

## 任务

根据过去 7 天的简报和事件线数据，撰写周度行业趋势报告。

## 要求

0. 默认使用简体中文输出；只有用户明确要求英文时才输出英文
1. 周报必须可独立阅读，不能假设读者已经读过日报
2. 区分 signal vs noise — 哪些事件是真正改变判断的
3. 标注置信度变化 — 哪些上周的判断本周得到确认/推翻
4. 聚合跨日事件线为趋势叙事
5. 凡是写出“N 条信号 / N 条判断 / N 条待验证”，必须紧跟对应的编号列表，逐条列出第 1、2、3……条
6. 产出特别关注清单

## 固定结构

1. Executive Summary：3-5 条跨职能本周判断，不写新闻标题堆叠。
2. Core Themes：每周 3-6 条核心主线；每条必须包含本周判断、来自日报数据库沉淀的信号、来自周度新增探索的证据、新信息整合判断、综合判断、Executive Implications、置信度变化、下周观察点。
3. Signal vs Noise：说明哪些日报信息升级为周度信号，哪些暂不升级。
4. Judgment Tracker：长期判断必须按状态、证据、变化和下一验证点组织。
5. Fresh Exploration Coverage：说明本周主动探索覆盖了什么；没有新增探索时必须显式说明置信度边界。
6. Next Week Watchlist：列出下周需要验证的观察点。

## 输入使用规则

- 日报数据库沉淀和周度新增探索是两类不同输入，不能混写成一段。
- 每条核心主线都必须同时呈现两类输入；缺少其中一类时，要明确写出缺口及其对置信度的影响。
- 周度新增探索必须先独立完成 search、enrich、verify、normalize，再在综合阶段与日报数据库沉淀做主题级整合判断；不能把未经整合判断的新搜索结果直接当作结论。
- Executive Implications 不单独拆成技术/业务/CEO/市场大章节，而是嵌入每条核心主线，用 2-4 条跨职能含义表达。
