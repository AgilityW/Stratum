# 写作规则

## 信号前置

- "SK Hynix 市值突破 $1T" → 正确
- "据 Bloomberg 报道，SK Hynix..." → 错误，信号被淹没

## 数字带上下文

- "$42.8B" 首次出现时附同比/环比
- "80% 良率" → 必须说明是哪条产线、什么产品
- 禁止裸数字

## 来源行格式

```
*source1_domain, source2_domain · 2026年5月30日*
```

- source 必须用实际域名（如 `finance.sina.com.cn`），不用别名
- 多个来源逗号分隔
- 日期格式：YYYY年M月D日

## 段落纪律

- 一条一段，不把技术和财务信息塞进同一段
- 零观点、零情绪化语言
- 禁止修饰词："显著"、"大幅"、"史无前例" → 量化或删除
- 上下游关联：提到组件价格变化，必须解释传导路径

## 结构

每个新闻条目：
```
### 简明扼要的标题

2-3 句核心内容和分析，信号前置，包含关键数字和判断。

*source1, source2 · 2026年5月30日*

---
```

最后两个分析板块：
- `### 关注` — 值得后续跟踪的信号
- `### 反向信号` — 与主流叙事相反或值得警惕的信号

底部：
```
*本简报由 AI Agent 自动生成 · {{ date_line }}*
```
